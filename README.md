# Lattice-Swarm: Olfati-Saber Flocking

A clean, from-scratch Python/NumPy implementation of the three distributed
flocking algorithms from:

> **Olfati-Saber, R. (2006).** *Flocking for Multi-Agent Dynamic Systems:
> Algorithms and Theory.* IEEE Transactions on Automatic Control, 51(3),
> 401–420.

All six simulation results from the paper (Fig. 7 – Fig. 12) are reproduced
by running the demo scripts in this repo.

---

## What's Implemented

**Math primitives (Sec. II):**
- σ-norm and its gradient (eq. 8–9)
- smooth bump function ρ_h (eq. 10)
- uneven-sigmoid action φ and pairwise potential ψ_α (eq. 15–16)
- spatial adjacency a_ij and repulsive φ_β (eq. 11, 56)

**Three distributed algorithms:**
- **Algorithm 1** — (α,α) protocol (eq. 23). *Fragments* in general.
- **Algorithm 2** — adds γ navigational feedback (eq. 24). Produces true flocking.
- **Algorithm 3** — adds β obstacle-avoidance via virtual agents (eq. 58–59).

**Obstacle primitives (Lemma 4):**
- `SphereObstacle(y, R)` — spherical obstacles
- `WallObstacle(y, a)` — infinite-plane walls
- `GammaAgent(q0, p0, f_r)` — dynamic navigation target

**Diagnostics (Sec. IX.A):**
- Relative connectivity C(t) = rank(L) / (n-1)
- Cohesion radius R(t)
- Normalized velocity mismatch K̃(v)
- Normalized deviation energy Ẽ(q)

## Core API

```python
from flocking import FlockSim, SphereObstacle, GammaAgent
import numpy as np

# 100 agents starting at random, no velocity
q0 = np.random.randn(100, 2) * 30
p0 = np.zeros_like(q0)

# Algorithm 2: flock with a dynamic navigation target
p_d = np.array([5.0, 0.0])
sim = FlockSim(
    q0, p0,
    algorithm=2,
    target_pos=lambda t: p_d * t,
    target_vel=p_d,
    c1_alpha=3.0, c1_gamma=0.3,
)

for step in range(600):
    sim.step(dt=0.02, t=step * 0.02)

print(sim.cohesion_radius(), sim.relative_connectivity())
```

**Algorithm 3 with obstacles:**
```python
obstacles = [SphereObstacle((100, 20), 10.0),
             SphereObstacle((130, -20), 5.0)]
sim = FlockSim(q0, p0, algorithm=3, obstacles=obstacles, ...)
```

**Custom γ-agent dynamics:**
```python
def leader_dynamics(q, p):
    return np.array([0.0, np.sin(0.1 * q[0])])   # sinusoidal heading

gamma = GammaAgent(q0=np.zeros(2), p0=np.array([3., 0.]), f_r=leader_dynamics)
sim = FlockSim(q0, p0, algorithm=2, target_pos=gamma, ...)
```

## Parameters

Paper Sec. VIII reference values (used throughout):

| Symbol | Value | Meaning |
|---|---|---|
| d | 7.0 | desired lattice spacing |
| r | 1.2 d | interaction range (ratio κ = 1.2) |
| d' | 0.6 d | obstacle safety distance |
| r' | 1.2 d' | obstacle interaction range |
| ε | 0.1 | σ-norm parameter |
| a, b | 5, 5 | action function shape (⇒ c = 0) |
| h (α) | 0.2 | bump cutoff for φ_α |
| h (β) | 0.9 | bump cutoff for φ_β |
| dt | 0.02 – 0.03 s | integration step |

Control gains can be passed to `FlockSim` (`c1_alpha`, `c1_beta`, `c1_gamma`,
and their damping counterparts `c2_*`). Defaults satisfy `c2_ν = 2√c1_ν`.

## Mathematical Overview

Each α-agent `i` applies the control input

```
u_i  =  c1_α · Σ_{j∈N_i} φ_α(‖q_j − q_i‖_σ) · n_ij        (gradient / lattice)
      + c2_α · Σ_{j∈N_i} a_ij(q) · (p_j − p_i)              (velocity consensus)
      + c1_β · Σ_{k∈N_i^β} φ_β(‖q̂_ik − q_i‖_σ) · n̂_ik      (obstacle repulsion)
      + c2_β · Σ_{k∈N_i^β} b_ik(q) · (p̂_ik − p_i)           (obstacle damping)
      − c1_γ · (q_i − q_r) − c2_γ · (p_i − p_r)             (γ navigation)
```

- Algorithm 1 keeps only the first two terms.
- Algorithm 2 adds the γ terms.
- Algorithm 3 adds the β terms as well.

β-agents are virtual "ghost" agents at the projection of each α-agent onto
the boundary of nearby obstacles (Lemma 4). The action functions φ_α and φ_β
are built so the resulting force field is smooth, bounded, and has a finite
cut-off at `r` (respectively `d'`).
